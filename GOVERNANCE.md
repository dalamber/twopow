# Project Governance

## Roles

- **Maintainer:** reviews and merges changes, cuts releases, sets direction.
- **Contributors:** anyone who submits a pull request or issue.

## Decision Making

Routine changes are approved by maintainer review. Substantial changes — notably
any proposal to extend beyond base 2 — require an issue, a design discussion, and
explicit maintainer sign-off.

## Releases

Releases follow Semantic Versioning and are cut by tagging `vX.Y.Z`, which
triggers the release workflow.
